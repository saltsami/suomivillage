Hyvä. Lyödään heti “Suomi-kylä” käyntiin niin, että kontentti syntyy eventeistä + suhteiden/maineen kehittymisestä eikä sun käsin kirjoittamisesta.

Kickoff Pack v0.1 – Suomi-kylä + Some + Chat + Uutiset
1) Ydinsääntö: maailma tuottaa totuuden, media tuottaa tarinan

World/Event Bus = totuus (kuka teki mitä, missä, kenelle, seuraukset)

NPC Agents = teot + reaktiot + muistot

Media Layer = feed/chat/news, joka renderöi eventit eri näkökulmista
→ sama eventti synnyttää 3 sisältöä: “minäkuva”, “juoru”, “uutinen”.

2) Data-malli (minimi joka riittää autonomisuuteen)

Tee nämä taulut/collections:

entities

id, type (npc/place/item), name, meta

npc_profiles

npc_id

traits: OCEAN (0–1), risk (0–1), empathy (0–1), dominance (0–1)

values: community, fairness, status, money, freedom (0–1)

voice: slang_level, politeness, sarcasm, verbosity

roles: influencer | gossip | mediator | journalist | worker

posting: base_rate/day, defensiveness, bragging

relationships

from_npc, to_npc

trust (-100..100), respect, affection, jealousy, fear

grievances[], debts[], last_interaction_ts

events (append-only)

id, ts, place_id, type, actors[], targets[]

publicness (0..1), severity (0..1)

tags[], payload (json)

memories

npc_id, event_id, importance (0..1), summary, embedding

(plus) belief_updates (optional)

goals

npc_id, horizon (short/long), priority, goal_json, status

content

id, source_event_id

channel (FEED/CHAT/NEWS), author_id

text, tone, visibility

moderation_ok, publish_ts

3) Agent loop (tällä saat “tekemistä” ilman käsinjuonta)

LLM vain päätöksiin & tekstiin. Rutiinit + liike deterministisesti.

Pseudolooppi per NPC (event-aktivoitu + ajastettu):

Perception: lähialueen eventit + oma tila + top-relationship muutokset

Retrieve: hae 3–8 relevanttia muistoa (sama paikka/henkilö/tagi)

Intent: valitse 1–2 intenttiä (help, avoid, confront, impress, earn…)

Plan: valitse action-sarja (max 1–3 askelta)

Act: kutsu Game API (move/talk/trade/borrow/post…)

Write memory: tallenna eventti + päivitä suhteet

Optional post: jos (impact + persona + publicness) ylittää kynnyksen → tee FEED/CHAT-viesti

Budjetointi: NPC ajattelee vasta kun (a) relevantti event, (b) “social window” (esim. kahvioilta), tai (c) 15–60 min sim-aika tick.

4) “Media Layer” – se mikä tekee tästä seurattavan
Impact-score (mikä on julkaisemisen arvoista)

Laske eventille:

impact =
  0.30*novelty +
  0.25*conflict +
  0.20*publicness +
  0.15*status_of_people +
  0.10*cascade_potential


FEED: authorin oma intressi (brändi/puolustus/kehuskely)

CHAT: reaktiot + juorut (nopea, kärjistyvä)

NEWS: vain top 1–3 / “tunti”, plus iltakooste

Juoru-moottori (autonomia boost)

Kun event tapahtuu:

todistajat saavat “fact”

osa luo rumor-eventin (distortion 0–0.6) riippuen:

gossip-role, jealousy, fear, loyalty

rumor-eventit aiheuttaa uusia konflikteja (väärinkäsitys = ilmaista draamaa)
