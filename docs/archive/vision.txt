Hyvä. Tehdään moderni Suomi-kylä, jossa “todellisuus” on simulaatio ja yleisö näkee kaiken kylän omana somena + chattina + uutisina. Se on rakennettavissa niin, että draama syntyy automaattisesti ja on silti kontrollissa.

0) Konsepti yhdellä lauseella

Kylä pyörii eventeillä (kuka teki mitä, missä, kenelle) → sisältökerros muuntaa ne feediksi/chatiksi/uutisiksi → ihmiset seuraa kuin “live-saippuaa”.

1) Kanavat (UI/tuote) – mitä yleisö näkee

KyläFeed (Instagram/TikTok-tyylinen)

NPC-postaukset: kuulumiset, kuvat (voi olla myöhemmin), “päivän fiilis”, piikittely

Kommentit: muut NPC:t + (valvottu) yleisö

Toriparlamentti (WhatsApp/Discord-tyylinen ryhmächat)

nopea reagointi, juorut, kärjistykset, sovittelut

KyläUutiset (paikallislehden live)

toimitusagentti tekee otsikon + 2–5 riviä + “taustat”

iltakooste + “huomisen odotukset”

Tärkeä: sama tapahtuma näkyy eri tavalla: feed on “minäkuva”, chat on “reaktio”, uutiset on “kuratoitu totuus”.

2) Kylän “tekemisen moottori” (ettei tule tylsää)
Paikat, jotka pakottaa kohtaamisia (MVP)

S-Market / kyläkauppa

Kahvio / huoltsikka

Tori + kirppis

Kyläsauna / avantopaikka

Urheilukenttä / lätkäkoppi

Kyläyhdistyksen talo (kokoukset + talkoot)

Päivärunko (ei LLM)

aamurutiini → työ → “pakollinen kohtaamispaikka” → harrastus → ilta (sauna/kahvio)
Tämä takaa törmäykset ja muistot, vaikka LLM olisi pois päältä.

3) Draamamoottori (mikä oikeasti luo vihoja, suhteita ja kaaria)
Niukkuudet (kitka)

Raha / velat

Status (“kylän vaikuttaja”, “luotettava”, “huijari”)

Aika (lupaukset pettää)

Huomio (kuka hengaa kenen kanssa)

Resurssit (toripaikka, talkooapu, peräkärry, mökkivuoro)

Suhdearvot (numeeriset, LLM vain tulkitsee)

NPC A ↔ NPC B:

trust, respect, affection, jealousy, fear + grievances (lista) + debts (lista)

Juorumylly

Eventistä syntyy huhuversio todennäköisyydellä:

juoruilija > introvertti

kateus = liioittelu, pelko = demonisointi, lojaalisuus = puolustelu

4) Sisältökerros (se “kova somekanava”)
Kolme generaattoria

Post Generator (NPC:n ääni)

Chat Reactor (ryhmächat-reaktiot, one-linerit, kinastelu)

Newsroom (toimittaja valitsee top-eventit impact-scorella)

Impact-score (mikä päätyy uutisiin / trendaamaan)

novelty + conflict + publicness + status_of_people + cascade_potential
→ vain top-1% tapahtumista saa “uutisnoston”, muuten tulee spämmiä.

Julkaisusäännöt (jotta sisältö tuntuu oikealta)

Kaikki ei postaa kaikesta (roolitus).

Postaus on usein self-serving (maineen hallinta).

Uutiset pyrkii neutraaliin, mutta voi “klikkiytyä” hienovaraisesti.

Miksi nämä toimii: niillä on valmiiksi ristikkäiset intressit (resurssit, status, totuus vs. imago, juoru vs. rauha).

6) Ensimmäiset 3 tarinakaarta (valmiiksi suunniteltu)

Toripaikkasota: kuka sai parhaan paikan ja miksi → huhut, kateus, “todisteet”

Talkookriisi: sovittu talkoopäivä hajoaa → syylliset, sovittelut, boikotit

Mökkivuoro-mysteeri: avain “katoaa” → syytökset → paljastus (tai väärä)

Nämä ei ole käsikirjoitusta, vaan tilanneinjektoreita, joista agentit tekevät oman sotkunsa.

8) MVP-roadmap (nopea mutta oikein)

Vaihe 1 (1–2 viikkoa dev-aikaa, jos fokus):

simulaatio + event-bus + 12 NPC rutiinit

feed + chat + news UI (vaikka pelkkä web)

impact-score + newsroom

perusmoderointi + rate limits

Vaihe 2:

huhumoottori + maine/suhdearvot näkyviin “profiilikortteihin”

“päivän kooste” + “huomisen agenda”

yleisöäänestys 1x/päivä (vain parametreihin, ei suoraa tarinan pakotusta)

Vaihe 3:

kuvat (AI tai “stock”) + NPC:n “selfie-tyyli”

replay-aikajana + binge

kausiteemat (joulu, hirviviikot, kesätori, lakkouhka…)

Jos haluat seuraavaksi konkreettisen speksin, teen sulle heti:

12 NPC:n profiilikortit (persoonallisuusparametrit, harrastukset, postauskäytös, triggerit)

50 eventtiä (mikro) + 12 isoa makroeventtiä

julkaisusäännöt (milloin feed/chat/news laukeaa)
…niin että kylä starttaa “päivä 1” jo kiinnostavana.
